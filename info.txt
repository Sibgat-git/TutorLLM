SECTION 5: FULL RIGOROUS AUDIT — FINDINGS & ISSUES
------------------------------------------------------------
Audit conducted 2026-05-04. Tests derived from 7 first-principles
properties required for a gold-standard tutoring training dataset.

The 7 Required Properties:
  P1. Self-contained    — answerable without external context
  P2. Unambiguous       — exactly one defensible correct answer
  P3. Textual integrity — readable, no broken notation or OCR garbage
  P4. Format consistency — consistent structure for model training
  P5. Label correctness  — correct answer is actually correct
  P6. Representational balance — no catastrophic subject/difficulty skew
  P7. Explainability    — permits step-by-step derivation (tutor goal)

--- CRITICAL ISSUES (must fix before explanation generation) ---

[C1] UNICODE NORMALIZATION BUG IN SUBJECT CLASSIFIER
  Severity: CRITICAL
  Property violated: P6 (subject classification wrong for 743 questions)
  Finding: 743 questions are misclassified into "other/" despite belonging
           to physics/chemistry/math/biology. Root cause: the JSON data
           stores Bengali text in NFD Unicode form (e.g., ড + ◌় separately)
           while regex patterns in 3.subject_organize.py are NFC (ড় as
           single codepoint U+09DC). repr() of both looks identical in terminal
           but byte-level comparison fails, so regex never matches.
  Evidence: re.search('চল তড়িৎ', subject_field) → False
            re.search(NFC('চল তড়িৎ'), NFC(subject_field)) → True
  Fix: add unicodedata.normalize('NFC', ...) to both pattern and subject
       field in classify() function before matching.
  Impact: ~743 questions misassigned, subject counts for physics/chemistry
          are understated by ~400–500 each.

[C2] BROKEN STEM MATH NOTATION
  Severity: CRITICAL
  Property violated: P3, P7 (can't generate correct explanations)
  Finding: 338 / 2,613 STEM questions (12.9%) have lost mathematical
           structure during scraping/OCR. Fractions, superscripts, and
           inverse function notation are all gone.
  Examples:
    dy/dx     → dydx
    E/2       → E2
    sin⁻¹(3/5) → sin-1(35)
    x/(5y+cosy) → x5y+cosy
    5y² + sin y = x² → 5y2+siny=x2
  Breakdown: math=177, chemistry=91, physics=70
  Fix: these need human review or removal. Cannot be auto-corrected
       reliably — the original LaTeX/formula source is lost.

[C3] CONTEXT-DEPENDENT QUESTIONS (missing stimulus)
  Severity: CRITICAL
  Property violated: P1 (not self-contained)
  Finding: 43 questions reference a passage, paragraph, or stimulus
           ("উদ্দীপক", "read the passage", "according to the above")
           that is NOT stored in the question record. Without the
           stimulus, neither a student nor a model can answer.
  Breakdown: english=28, bangla=8, other=3, general_knowledge=2,
             physics=1, accounting=1
  Fix: remove all 43 from gold standard.

[C4] CONFLICTING LABEL QUESTIONS
  Severity: CRITICAL
  Property violated: P5 (label correctness)
  Finding: 10 questions have identical question_text but different
           correct_answer values. All are over-generic questions like
           "Choose the correct sentence." or "Which of the following
           is incorrect?" — they have no real identity and conflicting
           ground truth will directly poison model training.
  Fix: remove all 10 from gold standard.

--- HIGH PRIORITY ISSUES (fix before training) ---

[H1] NEAR-DUPLICATE QUESTIONS (post-normalization)
  Severity: HIGH
  Property violated: P2, train/test contamination risk
  Finding: After stripping punctuation and normalizing whitespace/case,
           126 additional duplicate pairs exist beyond the exact-text
           deduplication already done. These will cause train/test
           contamination if splits are done naively.
  Fix: apply normalized deduplication; keep most-recent exam year version.

[H2] ERROR-DETECTION FORMAT (options are letter labels)
  Severity: HIGH
  Property violated: P4, P7 (different task type, different explanation format)
  Finding: 59 questions where options = ['A','B','C','D'] and correct_answer
           is a letter. These are "find the error in the underlined part"
           questions — a valid type but structurally incompatible with the
           standard MCQ explanation format.
  Example: "Scarcely had [A]the airport when [B]the announcement was made
            [C]that the flight has been [D]cancelled."  Answer: D
  Fix: separate into its own category. Needs its own explanation template
       ("The error is in part D because...").

[H3] NUMBER FIELD WHITESPACE CONTAMINATION
  Severity: MEDIUM (cosmetic but affects data cleanliness)
  Property violated: P3
  Finding: Every record has dirty whitespace in the "number" field:
           "number": "1\n                        "
  Fix: apply .strip() across all records when building final dataset.

--- MEDIUM PRIORITY ISSUES ---

[M1] ALL/NONE-OF-ABOVE OPTIONS
  Severity: MEDIUM
  Property violated: P7 (tutor explanation breaks down)
  Finding: 77 questions include "all of the above" or "none of the above"
           as an option. In 15 of those cases, that IS the correct answer.
           A tutor explaining "the answer is all of the above" is nearly
           useless — the explanation must address each sub-claim.
  Fix: flag these records; generate extended explanation format that
       addresses all options.

[M2] ANSWER POSITION BIAS (D under-represented)
  Severity: MEDIUM
  Property violated: P6 (representational balance)
  Finding: Option D (index 3) is the correct answer only 20.3% of the time
           vs. the expected ~25% for uniform distribution. This may cause
           the fine-tuned model to develop positional preference against
           selecting D.
  Fix: document and monitor during training; consider stratified sampling
       if model shows positional bias during eval.

[M3] PRE-2000 ARCHAIC QUESTIONS
  Severity: MEDIUM
  Property violated: P7 (student-relevance)
  Finding: 525 questions (3.3%) are from 1989–1999. Bengali language and
           curriculum have changed significantly. These questions may use
           archaic vocabulary/spellings that confuse modern students.
  Fix: flag with exam_year for optional exclusion; at minimum, the tutor
       model should use contemporary language in explanations.

[M4] UNCLASSIFIED "OTHER" BUCKET — RECLASSIFIABLE SUBJECTS
  Severity: MEDIUM
  Property violated: P6 (subject balance distorted)
  Finding: After fixing [C1], ~847 questions remain in "other" with 200
           distinct subject labels. Many have identifiable subjects not
           covered by current classifier patterns:
             Economics (অর্থনীতি)             :  43
             Mixed-exam batches (সবগুলো বিষয়)  : 101
             Fine Arts (চারু কারুকলা)           :  21
             Double-space typos in label        :  ~15 (e.g., "জ্যামিতিক  আলোকবিজ্ঞান")
           Plus many sub-chapter labels that ARE physics/chemistry/math
           but failed regex due to parenthetical chapter numbers, e.g.,
           "গুণগত রসায়ন (দ্বিতীয় অধ্যায়)" — fix by using substring match.
  Fix: extend SUBJECT_RULES with partial-match patterns for chapter-labeled
       subjects; add Economics as new subject category.

--- MISSING ENTIRELY (required for paper / tutoring goal) ---

[MISSING-1] NO EXPLANATION FIELD
  This is the core gap. Every record has question + options + correct_answer
  but ZERO step-by-step explanations. The paper goal requires training a
  model to explain like a home tutor. Without explanations as ground truth,
  fine-tuning only produces a question-answering model, not a tutoring model.
  Plan: generate explanations via strong teacher model (Claude/GPT-4) using
        subject-specific prompts that enforce step-by-step student-level
        derivation in Bengali/bilingual format.

[MISSING-2] NO TOPIC FIELD (fine-grained)
  Current metadata has only broad subject (e.g., "physics").
  Need fine-grained topic like "Projectile Motion", "Gas Laws (PV=nRT)",
  "Implicit Differentiation", "Pronoun Agreement".
  Required for: curriculum alignment, difficulty estimation, explanation
  prompt construction, and paper claims about topic coverage.

[MISSING-3] NO DIFFICULTY FIELD
  No difficulty level (easy/medium/hard). A good tutor teaches in
  difficulty progression. Without this field the model has no curriculum
  signal to learn from.
  Proxy approach: combine exam_year (older = generally harder),
  question_text length, and subject-specific complexity heuristics.

[MISSING-4] METRIC 7 (AMBIGUITY) NEVER EVALUATED
  Questions with multiple plausible correct answers were flagged as needing
  AI/human evaluation but were never actually evaluated. Some of these
  passed into gold_standard unchanged. For language/literature questions
  especially, ambiguity is common (e.g., Bangla poetry interpretation).
  Plan: use LLM-as-judge to score each question for answer uniqueness
  before explanation generation begins.

[MISSING-5] QUESTION_TYPE SUBTYPE FIELD
  All records are labeled "mcq" but the actual cognitive type varies:
    - factual recall          ("Who wrote...?")
    - calculation/derivation  ("If 5y²+sin y=x², find dy/dx")
    - fill-in-blank           ("আপনার সত্যকে পথপ্রদর্শক কাণ্ডারি বলে জানা–")
    - error-detection         ("Find the error in the underlined part")
    - comprehension           ("According to the passage...")
  The explanation format must differ per type. This field is required
  for the explanation generation prompts to work correctly.
